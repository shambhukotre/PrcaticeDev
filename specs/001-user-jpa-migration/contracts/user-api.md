# API Contract: User Service REST Endpoints

**Feature**: User Service Database Migration  
**Date**: April 10, 2026  
**API Version**: v1  
**Base URL**: `http://localhost:8080/api/users`

---

## Overview

The User Service exposes REST endpoints for managing user records in the SmartCollab platform. This contract defines the request/response schemas, status codes, and error handling for all CRUD and search operations.

All endpoints:
- Accept/return JSON content type
- Support CORS from `http://localhost:4200` (Angular frontend)
- Include proper HTTP status codes
- Return consistent error response formats

---

## Endpoints

### 1. GET /api/users

**Description**: Retrieve all users

**Method**: GET

**Request**:
```http
GET /api/users HTTP/1.1
Host: localhost:8080
Accept: application/json
```

**Response**:

**Status Code**: `200 OK`

**Response Body**:
```json
[
  {
    "id": 1,
    "name": "Shambhu",
    "email": "shambhu@test.com"
  },
  {
    "id": 2,
    "name": "Rahul",
    "email": "rahul@gmail.com"
  }
]
```

**Response Schema**:
```json
{
  "type": "array",
  "items": {
    "$ref": "#/components/schemas/User"
  }
}
```

**Examples**:
```bash
# Get all users
curl http://localhost:8080/api/users

# Get with pretty print
curl http://localhost:8080/api/users | jq .

# Get with specific format
curl -H "Accept: application/json" http://localhost:8080/api/users
```

### 3. POST /api/users

**Description**: Create new user

**Method**: POST

**Request Headers**:
| Header | Value | Required |
|--------|-------|----------|
| Content-Type | application/json | Yes |

**Request Body Schema**:
```json
{
  "type": "object",
  "required": ["name", "email"],
  "properties": {
    "id": {
      "type": "integer",
      "description": "ID (auto-generated by server, ignored if provided)"
    },
    "name": {
      "type": "string",
      "description": "User display name",
      "minLength": 1
    },
    "email": {
      "type": "string",
      "format": "email",
      "description": "Valid email address"
    }
  }
}
```

**Request Example**:
```http
POST /api/users HTTP/1.1
Host: localhost:8080
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john@example.com"
}
```

**Response - Success**:

**Status Code**: `201 Created`

**Response Headers**:
```
Location: /api/users/17
```

**Response Body**:
```json
{
  "id": 17,
  "name": "John Doe",
  "email": "john@example.com"
}
```

**Response - Validation Error**:

**Status Code**: `400 Bad Request`

**Response Body**:
```json
{
  "timestamp": "2026-04-10T10:30:00.000Z",
  "status": 400,
  "errors": {
    "name": "Name is required",
    "email": "Email should be valid"
  },
  "path": "/api/users"
}
```

**Validation Rules**:
| Field | Rule | Error Message |
|-------|------|---------------|
| name | @NotBlank | "Name is required" |
| email | @NotBlank | "Email is required" |
| email | @Email | "Email should be valid" |

**Examples**:
```bash
# Create valid user
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"name":"John Doe","email":"john@example.com"}'
# Returns: 201 Created with id: 17

# Missing name (validation error)
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"email":"john@example.com"}'
# Returns: 400 Bad Request

# Invalid email (validation error)
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"name":"John","email":"invalid"}'
# Returns: 400 Bad Request

# ID provided (ignored)
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"id":999,"name":"Jane","email":"jane@example.com"}'
# Returns: 201 Created with id: 18 (auto-generated, not 999)
```

### 6. GET /api/users/search

**Description**: Search users by name and/or email

**Method**: GET

**Query Parameters**:
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| name | String | No | Partial name to search (case-insensitive) |
| email | String | No | Email pattern to search (case-insensitive) |

**Request Examples**:
```http
# Search by name
GET /api/users/search?name=raj HTTP/1.1
Host: localhost:8080

# Search by email
GET /api/users/search?email=gmail HTTP/1.1
Host: localhost:8080

# Search by both (OR logic)
GET /api/users/search?name=raj&email=gmail HTTP/1.1
Host: localhost:8080

# Search with empty query
GET /api/users/search HTTP/1.1
Host: localhost:8080
```

**Response**:

**Status Code**: `200 OK`

**Response Body** (search by name="raj"):
```json
[
  {
    "id": 9,
    "name": "Rajesh",
    "email": "rajesh@gmail.com"
  },
  {
    "id": 15,
    "name": "Sanjay",
    "email": "sanjay@gmail.com"
  }
]
```

**Search Logic**:
- If `name` parameter provided: Search in name field (LIKE with %)
- If `email` parameter provided: Search in email field (LIKE with %)
- If both provided: OR logic (match name OR email)
- If neither provided: Return all users

**Examples**:
```bash
# Search by name (partial, case-insensitive)
curl "http://localhost:8080/api/users/search?name=raj"
# Returns: Rajesh, Sanjay

# Search by name (different case)
curl "http://localhost:8080/api/users/search?name=RAJ"
# Returns: Rajesh, Sanjay

# Search by email domain
curl "http://localhost:8080/api/users/search?email=gmail"
# Returns: All users with gmail.com email

# Search by first name
curl "http://localhost:8080/api/users/search?name=amit"
# Returns: Amit

# Empty search (returns all)
curl "http://localhost:8080/api/users/search"
# Returns: All users
```

## HTTP Status Codes

| Status Code | Meaning | Use Case |
|-------------|---------|----------|
| 200 OK | Success | GET, PUT, DELETE successful |
| 201 Created | Resource created | POST successful, Location header set |
| 400 Bad Request | Validation error | Invalid email format, missing required fields |
| 404 Not Found | Resource not found | GET/PUT/DELETE non-existent user |
| 500 Internal Server Error | Server error | Unexpected exception |

## CORS Configuration

All endpoints support CORS requests from configured origins.

**Allowed Origins**: `http://localhost:4200` (Angular frontend)

**Allowed Methods**: GET, POST, PUT, DELETE

**Allowed Headers**: Content-Type, Accept

**Example CORS Request**:
```javascript
// From Angular frontend running on http://localhost:4200
fetch('http://localhost:8080/api/users', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
})
.then(response => response.json())
.then(data => console.log(data));
```
