# Quick Start Guide: User Service Database Migration

**Feature**: User Service Database Migration  
**Date**: April 10, 2026  
**Duration**: ~30 minutes to get started

---

## Pre-requisites

- JDK 17 or higher installed
- Maven 3.6+ installed
- IDE: IntelliJ IDEA, VS Code with Spring Boot extensions, or Eclipse
- Git (for version control)
- H2 Console (built-in, no install needed)

### Verify Installation

```bash
# Check Java version
java -version
# Should output: openjdk version "17.x.x"

# Check Maven version
mvn -version
# Should output: Apache Maven 3.6.x

# Navigate to project root
cd Backend/Backend
```

## Phase 0: Build & Setup (5 minutes)

### 1. Build the Project

```bash
cd Backend/Backend
mvn clean install
```

**Expected Output**:
```
[INFO] BUILD SUCCESS
[INFO] Total time: XX.XXX s
```

If you see dependency errors, run:
```bash
mvn dependency:resolve
```

### 2. Verify H2 Dependencies

Check that Spring Data JPA and H2 are in `pom.xml`:

```xml
<!-- Should be present -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

### 3. Start the Application

```bash
mvn spring-boot:run
```

**Expected Console Output**:
```
Starting BackendApplication v0.0.1-SNAPSHOT using Java 17.x.x
...
org.springframework.boot.Application Started BackendApplication in X.XXX seconds
```

✅ Application is running on `http://localhost:8080`

## Phase 1: Explore the API (5 minutes)

### 1. Get All Users

```bash
curl http://localhost:8080/api/users
```

**Response**:
```json
[
  {"id": 1, "name": "Shambhu", "email": "shambhu@test.com"},
  {"id": 2, "name": "Rahul", "email": "rahul@gmail.com"},
  ...
]
```

### 2. Get User by ID

```bash
curl http://localhost:8080/api/users/1
```

**Response**:
```json
{"id": 1, "name": "Shambhu", "email": "shambhu@test.com"}
```

### 3. Create New User

```bash
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"name":"John Doe","email":"john@example.com"}'
```

**Response**:
```json
{"id": 17, "name": "John Doe", "email": "john@example.com"}
```

Note: ID is auto-generated by the database.

### 4. Update User

```bash
curl -X PUT http://localhost:8080/api/users/17 \
  -H "Content-Type: application/json" \
  -d '{"name":"John Smith","email":"john.smith@example.com"}'
```

### 5. Search Users

```bash
# Search by name (partial match, case-insensitive)
curl "http://localhost:8080/api/users/search?name=raj"

# Expected: Returns "Rajesh", "Sanjay" (both contain "raj")

# Search by email
curl "http://localhost:8080/api/users/search?email=test.com"
```

### 6. Delete User

```bash
curl -X DELETE http://localhost:8080/api/users/17
```

**Response**: `true` (success) or `false` (user not found)

## Phase 2: Inspect H2 Database (5 minutes)

### 1. Access H2 Console

Open browser: `http://localhost:8080/h2-console`

### 2. Login

- **JDBC URL**: `jdbc:h2:file:./data/smartcollab`
- **User Name**: `sa`
- **Password**: (leave empty)

Click **Connect**

### 3. Explore Tables

```sql
-- See all tables
SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES;

-- View user table structure
DESCRIBE user;

-- Query all users
SELECT * FROM user;

-- Count total users
SELECT COUNT(*) FROM user;

-- Find users by email pattern
SELECT * FROM user WHERE email LIKE '%gmail.com%';
```

### 4. Verify Data Persistence

- Create a new user via API (see Phase 1, step 3)
- Refresh H2 Console query and verify new user exists
- Stop the application (Ctrl+C)
- Start again: `mvn spring-boot:run`
- Query H2 again: SELECT * FROM user;
- ✅ New user should still exist (data persisted!)

## Common Tasks

### Clean Build

```bash
# Remove all build artifacts
mvn clean

# Rebuild everything
mvn clean install

# Reset H2 database (removes ./data/smartcollab.db)
rm -rf ./data/smartcollab.db

# Restart application to recreate with seed data
mvn spring-boot:run
```

### Debug in IDE

**IntelliJ IDEA**:
1. Click on `BackendApplication.java`
2. Click **Run** → **Debug 'BackendApplication'**
3. Set breakpoints in UserService or UserController
4. Make API calls to hit breakpoints

**VS Code**:
1. Install "Spring Boot Dashboard" extension
2. Click the app in Dashboard
3. Click "Debug" icon
4. Same breakpoint process

### View Application Logs

```bash
# When running with mvn spring-boot:run, logs display in console

# For JAR file deployment:
java -jar target/Backend-0.0.1-SNAPSHOT.jar > logs/app.log 2>&1

# Watch logs in real-time
tail -f logs/app.log
```

### Database Queries Logging

Edit `application.properties`:
```properties
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
```

Restart application to see all SQL in console output.

## Troubleshooting

### Issue: Port 8080 Already in Use

**Solution**:
```bash
# Kill existing process on port 8080
# macOS/Linux:
lsof -ti:8080 | xargs kill -9

# Windows (PowerShell):
Get-Process -Id (Get-NetTCPConnection -LocalPort 8080 -ErrorAction SilentlyContinue).OwningProcess | Stop-Process -Force

# Or change port in application.properties:
server.port=8081
```

### Issue: H2 Database File Locked

**Symptom**: "Database may be already in use"

**Solution**:
```bash
# Stop the application
# Wait 5 seconds
# Delete database file
rm -rf ./data/smartcollab.db

# Restart application
mvn spring-boot:run
```

### Issue: Maven Compilation Error

**Solution**:
```bash
# Clear Maven cache
rm -rf ~/.m2/repository

# Reinstall dependencies
mvn clean install -U

# If still fails, check Java version
java -version  # Must be 17+
```

### Issue: Validation Not Working

**Symptom**: Invalid data accepted when it shouldn't be

**Solution**:
1. Verify `spring-boot-starter-validation` in pom.xml
2. Check User.java has `@NotBlank`, `@Email` annotations
3. Check UserController has `@Valid` annotation
4. Rebuild: `mvn clean package`

### Issue: Tests Fail

**Solution**:
```bash
# Check specific test output
mvn test -Dtest=UserServiceTest -e

# Run with debug output
mvn test -X

# Skip tests to verify build works
mvn clean install -DskipTests
```
